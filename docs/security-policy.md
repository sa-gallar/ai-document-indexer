Данный документ определяет границы ответственности разработчиков системы `AI Document Indexer` в отношении обработки персональных данных (ПДн), конфиденциальной информации и коммерческой тайны.

## ⚖️ Ограничение ответственности (Disclaimer)

1. **Легитимность источников данных:** Система не осуществляет сбор, поиск или предоставление архивных документов. Пользователь самостоятельно загружает файлы в приложение. Способ, законность и правомерность получения данных файлов пользователем находятся целиком вне зоны ведения и контроля разработчиков системы.
2. **Отсутствие встроенной защиты ПДн:** В системе не предусмотрены встроенные механизмы маскирования, деперсонализации или защиты персональных данных, коммерческой тайны или иных видов конфиденциальной информации. 
3. **Ответственность за комплаенс:** Вся ответственность за соблюдение законодательства о персональных данных (включая Федеральный закон № 152-ФЗ или аналогичные международные акты), законов о коммерческой и государственной тайне при работе с документами целиком и полностью возлагается на конечного пользователя (архивиста / исследователя).

## 🔒 Технические особенности обработки данных

Для понимания рисков безопасности пользователю необходимо учитывать архитектуру сквозного процесса:
* **Передача данных во внешние контур:** Конвейер обработки включает в себя передачу текстового и графического слоя документов во внешние API-интерфейсы моделей ИИ (LLM / Vision API) сторонних провайдеров.
* **Рекомендация для пользователей:** Настоятельно не рекомендуется загружать в систему документы, содержащие актуальные персональные данные живых лиц, конфиденциальную корпоративную информацию, коммерческую тайну или документы с действующими грифами ограничения доступа.

## 🔑 Безопасность API-ключей и учетных данных (API Key Management)

Поскольку конвейер обработки данных использует внешние модели ИИ через API-интерфейсы, разработчики и пользователи обязаны строго соблюдать правила информационной безопасности:

1. **Запрет на жесткое кодирование (Hardcoding):** Категорически запрещено записывать API-ключи, токены доступа или пароли внутри исходного кода приложения (`.py`, `.js` и т.д.).
2. **Использование переменных окружения:** Все конфиденциальные данные (API-ключи) должны считываться приложением исключительно из переменных окружения (Environment Variables) системы или из локального файла конфигурации **`.env`**.
3. **Защита репозитория (.gitignore):** Локальный файл `.env`, содержащий реальные ключи доступа, должен быть в обязательном порядке внесен в файл исключений `.gitignore`, чтобы предотвратить его случайную публикацию в публичный репозиторий GitHub.

## 📖 Инструкция для пользователя: Как получить API-ключ Gemini (Google AI Studio)

Для работы программы вам понадобится личный бесплатный ключ доступа к искусственному интеллекту Google Gemini. Чтобы получить его, выполните следующие шаги:

1. **Перейдите на сайт разработчиков:** Откройте в браузере страницу **[Google AI Studio](https://google.com)** и войдите под своим обычным Google-аккаунтом.
2. **Создайте ключ:** В левом верхнем углу нажмите на большую синюю кнопку **"Get API key"** (Получить API-ключ).
3. **Сгенерируйте токен:** Нажмите кнопку **"Create API key"**, выберите в списке проект по умолчанию и подтвердите создание.
4. **Скопируйте ключ:** Перед вами появится длинная строка из букв и цифр (например, `AIzaSy...`). Нажмите кнопку **"Copy"** рядом с ней.
5. **Активация в программе:** Создайте в папке с программой простой текстовый файл с именем `.env` и вставьте туда скопированный ключ в следующем формате:
   ```text
   GEMINI_API_KEY=ваш_скопированный_ключ_сюда
   ```
