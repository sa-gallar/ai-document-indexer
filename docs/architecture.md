## 🗺️ Архитектура процесса (System Workflow)

Ниже представлена UML-диаграмма последовательности, описывающая сквозной процесс обработки документов от выбора папки до генерации классификаторов:

```mermaid
sequenceDiagram
    autonumber
    actor User as Архивист / Исследователь
    participant App as Программа (AI Document Indexer)
    participant AI as Модель ИИ (LLM / Vision API)
    participant DB as Буфер данных (Память)

    User->>App: Указывает папку/источник с файлами
    App->>App: Сканирует папку и формирует очередь файлов
    
    loop Для каждого графического файла (скан, фото, страница)
        App->>AI: Отправляет файл (Запрос: Распознать текст + учесть язык)
        Note over AI: OCR-сканирование +<br/>Семантический анализ контекста
        AI->>App: Возвращает структурированные сущности (ФИО, Адрес, Метаданные: дата, должность)
        App->>DB: Временно сохраняет сырые извлеченные данные
    end

    App->>AI: Запрос: Систематизировать и агрегировать собранные данные
    Note over AI: Классификация, дедупликация<br/>и алфавитная сортировка
    AI->>App: Возвращает финальные структуры классификаторов
    App->>User: Выгружает отчеты (Фамильный и Адресный указатели в Excel/Markdown)
```
